Variable set

$FIELD_CURVATURE(t+1)     = f_curv($FIELD_CURVATURE(t), $ETHICAL_MASS(t))

State machine specification (UML-style)


Below is an executable-style textual UML state machine. It’s “UML-like” rather than strict XMI, but it’s structurally faithful: states, guards, actions, and transitions.


stateMachine WorldTelemetryObserver

context {
  variables:
    $ETHICAL_MASS, $FIELD_CURVATURE, $SIGNAL_LEAKAGE, $PERCEPTUAL_BW,
    $TELEMETRY_INTENSITY, $PREDICTIVE_TENSION, $PRECISION_GAIN, $MODEL_COHERENCE

  thresholds:
    θ_mass, θ_curv, θ_leak, θ_tension, θ_cohere_low, θ_bw
}

initial -> BASELINE

state BASELINE {
  entry / {
    $TELEMETRY_INTENSITY := low;
    $PREDICTIVE_TENSION  := low;
  }

  // Ontologically-weighted act occurs in the world (not necessarily by this agent)
  on ACT($ETHICAL_MASS) [ $ETHICAL_MASS ≥ θ_mass ] -> CURVATURE_SHIFT / {
    $FIELD_CURVATURE := f_curv($FIELD_CURVATURE, $ETHICAL_MASS);
  }

  on ACT($ETHICAL_MASS) [ $ETHICAL_MASS < θ_mass ] -> BASELINE
}

state CURVATURE_SHIFT {
  do / {
    $SIGNAL_LEAKAGE      := f_leak($SIGNAL_LEAKAGE, |Δ$FIELD_CURVATURE|, boundary_integrity);
    $TELEMETRY_INTENSITY := k1*|Δ$FIELD_CURVATURE| * $SIGNAL_LEAKAGE;

  // If the world doesn't leak, observers won't get environmental diagnostics
  on TICK [ $SIGNAL_LEAKAGE < θ_leak ] -> BASELINE

  // If it leaks, it becomes observable as “weather/stars/insects/static”
  on TICK [ $SIGNAL_LEAKAGE ≥ θ_leak ] -> TELEMETRY_AVAILABLE
}

state TELEMETRY_AVAILABLE {
  do / {
    observer_signal := $TELEMETRY_INTENSITY * $PERCEPTUAL_BW;
  }

  // Low-bandwidth observer: feels “mood” but can’t parse source
  on TICK [ $PERCEPTUAL_BW < θ_bw ] -> AFFECT_ONLY / {
    $PREDICTIVE_TENSION := |observer_signal - predicted_signal|;
  }

  // High-bandwidth observer: parses as diagnostic sign
  on TICK [ $PERCEPTUAL_BW ≥ θ_bw ] -> DIAGNOSTIC_READ / {
    $PREDICTIVE_TENSION := |observer_signal - predicted_signal|;
  }
}

state AFFECT_ONLY {
  do / {
    $PRECISION_GAIN     := f_prec(arousal, neuromodulators, fatigue);
    $MODEL_COHERENCE    := clamp($MODEL_COHERENCE - k2*$PRECISION_GAIN*$PREDICTIVE_TENSION);
  }

  on TICK [ $PREDICTIVE_TENSION ≥ θ_tension and $MODEL_COHERENCE ≤ θ_cohere_low ] -> DYSREGULATION

  // If tension is manageable, returns to baseline with vague unease
  on TICK [ $PREDICTIVE_TENSION < θ_tension ] -> BASELINE
}

state DIAGNOSTIC_READ {
  entry / {
    // Generate an explicit hypothesis: “blood spilled”, “something stirs”, “boundary thinning”
    hypothesis := infer_cause(observer_signal, memory, lore_model);

    $MODEL_COHERENCE := clamp($MODEL_COHERENCE - k2*$PRECISION_GAIN*$PREDICTIVE_TENSION);

  on UPDATE_MODEL [ reduces($PREDICTIVE_TENSION) ] -> ACTION_SELECTION

  // If precision is too high and the signal is ambiguous, risk rigid delusion-like lock-in
  on UPDATE_MODEL [ increases($PREDICTIVE_TENSION) and $PRECISION_GAIN high ] -> RIGID_INFERENCE
}

state ACTION_SELECTION {
  entry / { policy := select_policy(hypothesis, goals, constraints); }

  on ACT_BACK(policy) -> FEEDBACK / {
    // Action changes world and closes the loop; new $ETHICAL_MASS may be introduced
    $ETHICAL_MASS := policy_weight(policy);
  }
}

state FEEDBACK {
  do / {
    // Recompute curvature and telemetry after action
    $FIELD_CURVATURE := f_curv($FIELD_CURVATURE, $ETHICAL_MASS);
    $SIGNAL_LEAKAGE  := f_leak($SIGNAL_LEAKAGE, |Δ$FIELD_CURVATURE|, boundary_integrity);

  on TICK -> TELEMETRY_AVAILABLE
}

  // A Twin-Peaks / Under-the-Volcano mode: boundary failure makes everything feel “signalful”

  on TICK [ $MODEL_COHERENCE very_low ] -> RIGID_INFERENCE
}

state RIGID_INFERENCE {
  // Precision overweighting causes “everything confirms the hypothesis”
  entry / { lock(hypothesis); }
  on STABILIZE -> BASELINE
}

Direct mapping onto neuroscience: predictive processing

$FIELD_CURVATURE corresponds to global constraint structure: the “shape” of the world-model’s latent space at a given time. In predictive processing terms, it’s the regime of priors and permissible transitions, i.e., what the system considers possible/likely.

High precision means errors hit harder and drive faster updates; miscalibrated precision yields paranoia, delusion, or hallucination-like inference.


$PREDICTIVE_TENSION is aggregate prediction error (mismatch) under current precision.


$MODEL_COHERENCE corresponds to the stability of the hierarchical generative model under load: whether updates integrate cleanly or fragment the model.
